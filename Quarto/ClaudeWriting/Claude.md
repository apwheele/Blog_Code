# Root CLAUDE.md - Writing Projects Hub

## Project Overview
This is the root directory for all writing projects. It contains multiple subdirectories for different types of content, each with their own style guidelines and examples.

## Project Structure

These sub-sections are all folders, each folder has example writings in txt, md, or qmd format. So go into the folders and read the relevant contents.

### BookWriting/
**Purpose**: Technical Python book content in Quarto (.qmd) format  
**Style**: Technical but accessible, progressive complexity, code-heavy  
**When to use**: Python tutorials, technical book chapters, educational programming content  
**Reference examples**: Look at existing .qmd files in BookWriting/ for tone, structure, and code style

### CrimeDeCoderBlog/
**Purpose**: Consulting website blog posts and service descriptions in Markdown (.md)  
**Style**: Professional but approachable, client-focused, forensics/litigation topics  
**When to use**: Blog posts about digital forensics, consulting services, case studies, content for attorneys  
**Reference examples**: Review .md files in CrimeDeCoderBlog/ for tone, technical depth, and client-facing language

### LinkedInPosts/
**Purpose**: Short-form professional social media content  
**Style**: Conversational professional, hook-driven, engagement-focused  
**When to use**: LinkedIn updates, thought leadership snippets, professional insights, quick tips  
**Reference examples**: Check LinkedInPosts/ for formatting, length, and engagement style

### PersonalTechBlog/
**Purpose**: In-depth technical blog posts on data science, Python, ML, forensics  
**Style**: Technical depth with personality, tutorial-oriented, opinionated, code-heavy  
**When to use**: Deep technical tutorials, tool reviews, implementation guides, technical commentary  
**Reference examples**: See PersonalTechBlog/ for technical depth, code integration, and personal voice

### Resume/
**Purpose**: Professional resume and CV materials  
**Style**: Concise, achievement-focused, quantified results  
**When to use**: Reference for biographical information, credentials, experience in other projects  
**Reference examples**: Use Resume/ for accurate dates, titles, accomplishments, technical skills

## Workflow Guidelines

### When Starting a New Writing Task:
1. **Identify the content type** - Which folder does this belong in?
2. **Review the folder's CLAUDE.md** - Understand specific style guidelines
3. **Check existing examples** - Look at 2-3 similar files in that folder
4. **Reference Resume/** - Pull biographical details if needed for context
5. **Match the voice and structure** - Maintain consistency with existing content

### Cross-Folder Context:
- Resume content can be referenced from any project for credentials
- BookWriting examples show my technical teaching style
- CrimeDeCoderBlog shows my professional consulting voice
- PersonalTechBlog shows my authentic technical writing voice
- Projects shows my formal business documentation style
- LinkedInPosts shows my social media engagement approach

### Example Commands:
- "Write a CrimeDeCoder blog post about [topic]" → Check CrimeDeCoderBlog/ examples first
- "Draft a LinkedIn post about [topic]" → Review LinkedInPosts/ for style and length
- "Create a proposal for [project]" → Reference Projects/ structure and Resume/ for qualifications
- "Write a tutorial on [topic]" → Choose between BookWriting/ (book chapter) or PersonalTechBlog/ (standalone post)

## Writing Style Principles (Across All Projects)

### Universal Guidelines:
- **Clarity first**: Complex ideas should be understandable
- **Active voice**: Prefer active constructions
- **Concrete examples**: Show, don't just tell
- **Audience awareness**: Adjust technical depth appropriately
- **Authentic voice**: Maintain personality while meeting professional standards

### Technical Content:
- All code must be tested and functional
- Include context for why, not just how
- Provide reproducible examples
- Cite sources and give credit

### Professional Content:
- Confidence without arrogance
- Evidence-based claims
- Transparent about limitations
- Client/reader value-focused

## Quality Standards

### Before Finalizing Any Content:
- Does it match the style of existing content in that folder?
- Is the technical depth appropriate for the audience?
- Are all facts, dates, and credentials accurate (check Resume/ if needed)?
- Is the tone consistent with the folder's guidelines?
- Would this content serve its intended purpose?

## Key Reminders
- **Different audiences require different approaches** - A LinkedIn post is not a blog post is not a proposal
- **Examples are your best guide** - Always review similar existing content
- **Resume is the source of truth** - For biographical details, check Resume/ first
- **Maintain voice consistency** - Each folder has an established voice to match
- **When in doubt, ask** - Clarify the target folder and audience before writing

## File Naming Conventions
- **BookWriting/**: chapterXX.qmd or descriptive-topic.qmd
- **CrimeDeCoderBlog/**: descriptive-title-with-hyphens.md, date-prefixed for posts
- **LinkedInPosts/**: YYYY-MM-DD-topic.txt or brief-description.txt
- **PersonalTechBlog/**: YYYY-MM-DD-descriptive-title.md
- **Projects/**: ClientName-ProjectType-Date.md or descriptive-project-name.md
