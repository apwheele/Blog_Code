# BookWriting/CLAUDE.md

## Project Overview

This folder contains Quarto (.qmd) files for a technical Python book (python_c0?.qmd) and a book on using LLM foundation models (llm_c0?.qmd). Each file represents a chapter or major section of the book. Use these as exemplars if writing technical notes, future books, or coding tutorials for blog posts.

## Writing Style

- **Technical but accessible**: Explain complex concepts clearly for intermediate Python developers
- **Progressive complexity**: Build from fundamentals to advanced topics within each chapter
- **Practical focus**: Every concept should include working code examples
- **Active voice**: Prefer "You can use this method" over "This method can be used"
- **Conversational authority**: Expert guidance delivered in a friendly, teaching tone

## Code Standards

- All code examples must be tested and functional
- Follow PEP 8 style guidelines
- Include comments for non-obvious logic
- Show both the code and expected output where relevant
- Use realistic variable names and examples

## File Structure
- Each .qmd file is a self-contained chapter when writing a book
- Code blocks use Python syntax highlighting

## Key Guidelines
- Maintain consistent voice and difficulty progression across chapters
- Avoid introducing concepts before they're explained
- Include "why" explanations, not just "how"
- Anticipate common mistakes and address them proactively
- End sections with practical applications or next steps
