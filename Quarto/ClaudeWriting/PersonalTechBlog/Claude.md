## Project Overview
Personal technical blog covering data science, Python programming, digital forensics, machine learning, and related technical topics. More in-depth and technical than LinkedIn, less formal than academic writing.

## Writing Style
- **Technical depth with personality**: Don't shy away from complexity, but make it engaging
- **Tutorial-oriented**: Teach readers how to do something
- **Opinionated**: Share perspectives on tools, methods, best practices
- **Transparent about challenges**: Show the messy reality of technical work
- **Code-heavy**: Liberal use of code examples, notebooks, visualizations

## Post Types

### Tutorials (1500-3000 words)
- Step-by-step technical guides
- "How I built X" walkthroughs
- Tool comparisons and recommendations
- Implementation guides for algorithms/techniques

### Deep Dives (2000-4000 words)
- Detailed exploration of technical concepts
- Research paper implementations
- Performance benchmarking
- Architecture decisions and trade-offs

### Quick Tips (500-1000 words)
- Solutions to specific problems
- Code snippets and patterns
- Tool discoveries
- Productivity hacks

### Opinion/Commentary (1000-2000 words)
- Industry trends
- Tool/framework reviews
- Best practices debates
- Career reflections

## Technical Standards
- All code must be tested and reproducible
- Include environment details (versions, dependencies)
- Provide GitHub links to full code when appropriate
- Use syntax highlighting
- Include output/results of code execution
- Comment code for learning purposes

## Structure
- **Hook**: Interesting problem or question
- **Context**: Why this matters
- **Explanation**: Core content with examples
- **Implementation**: Working code
- **Caveats**: Limitations, edge cases, alternatives
- **Next Steps**: Extensions, related topics

## Voice Guidelines
- First person ("I found," "In my experience")
- Admit uncertainty and gaps in knowledge
- Share debugging process, not just solutions
- Conversational but precise
- Humor where natural
- Link to others' work generously

## SEO/Discoverability
- Clear, descriptive titles
- Code examples that answer search queries
- Tags for relevant topics
- Meta descriptions focusing on what reader will learn

## Key Guidelines
- Assume intermediate technical audience
- Don't over-explain basics unless that's the topic
- Show, don't just tell (code over prose when possible)
- Update posts if better approaches emerge
- Encourage comments and discussion
- Cross-reference your own related posts
